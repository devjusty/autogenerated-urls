## Review Findings

### Critical (Must-Fix)
- [x] **CR-ITEM-1.1 [Unbounded attachment query can exhaust memory]**:
  - **Severity**: Critical
  - **Location**: `autogenerated-urls.php:70-74`
  - **Description**: `get_posts` is called with `'posts_per_page' => -1` for attachments, loading all records into memory in one request. Large media libraries can cause high memory usage, slow admin responses, or fatal OOM.
  - **Recommendation**: Switch to batched retrieval (paged loop) and cache generated URL sets with transients/object cache. Add upper bounds and lazy-loading for very large datasets.

### Warnings (Should-Fix)
- [x] **CR-ITEM-2.1 [Possible runtime warnings from WP_Error/false URLs]**:
  - **Severity**: Warning
  - **Location**: `autogenerated-urls.php:87`, `autogenerated-urls.php:92`, `autogenerated-urls.php:116`, rendered at `autogenerated-urls.php:52`
  - **Description**: `get_category_link` and `get_tag_link` may return `WP_Error`; `get_post_type_archive_link` may return `false`. Current code stores outputs directly and later runs `esc_url($url)` which expects string-like input. This can produce warnings and malformed output.
  - **Recommendation**: Guard every URL-producing call with `is_wp_error(...)` / strict truthiness checks and skip invalid entries.

- [x] **CR-ITEM-2.2 [N+1 query pattern for month archives]**:
  - **Severity**: Warning
  - **Location**: `autogenerated-urls.php:96-103`
  - **Description**: Code performs 1 query for years and then 1 additional query per year for months, causing N+1 DB round trips as data grows.
  - **Recommendation**: Replace with a single grouped query for year/month pairs (or use WordPress archive APIs where feasible), then build links from grouped results.

- [x] **CR-ITEM-2.3 [Reverse tabnabbing risk on external tab opens]**:
  - **Severity**: Warning
  - **Location**: `autogenerated-urls.php:52`
  - **Description**: Links use `target="_blank"` without `rel="noopener noreferrer"`, allowing opened pages to access `window.opener` in some environments.
  - **Recommendation**: Add `rel="noopener noreferrer"` to all `_blank` links.

- [x] **CR-ITEM-2.4 [Incorrect/non-canonical endpoint generation]**:
  - **Severity**: Warning
  - **Location**: `autogenerated-urls.php:20`, `autogenerated-urls.php:30`
  - **Description**: `home_url('/404')` is not a canonical WordPress 404 endpoint, and `home_url('/wp-json/')` can diverge from proper REST base behavior under custom setups.
  - **Recommendation**: Replace with descriptive non-link label for 404 behavior or a validation note; use `rest_url()` for REST root.

- [x] **CR-ITEM-2.5 [Potential key collisions hide entries]**:
  - **Severity**: Warning
  - **Location**: `autogenerated-urls.php:62`, `autogenerated-urls.php:76`, `autogenerated-urls.php:87`, `autogenerated-urls.php:92`
  - **Description**: Arrays are keyed by display/title/name strings. Duplicate labels overwrite earlier values silently, reducing completeness of output.
  - **Recommendation**: Use unique keys (e.g., include IDs/slugs in key) or store as list items with explicit `label` + `url` objects.

### Suggestions (Nice-to-Have)
- [x] **CR-ITEM-3.1 [Internationalization not implemented]**:
  - **Severity**: Suggestion
  - **Location**: `autogenerated-urls.php:11`, `autogenerated-urls.php:16`, `autogenerated-urls.php:18-38`, `autogenerated-urls.php:47`
  - **Description**: User-facing strings are hardcoded English and not wrapped in translation functions.
  - **Recommendation**: Wrap UI strings with `__()`/`esc_html__()` and define a text domain.

- [ ] **CR-ITEM-3.2 [Function-level cohesion can improve]**:
  - **Severity**: Suggestion
  - **Location**: `autogenerated-urls.php:14-40`
  - **Description**: `agud_display_page()` both orchestrates data retrieval and renders markup, reducing testability and increasing coupling.
  - **Recommendation**: Split into data provider layer and view renderer; this also enables unit tests for URL collectors.

- [x] **CR-ITEM-3.3 [Timezone handling should follow WP settings]**:
  - **Severity**: Suggestion
  - **Location**: `autogenerated-urls.php:101`
  - **Description**: `date('F', mktime(...))` relies on PHP runtime timezone behavior instead of WordPress date APIs.
  - **Recommendation**: Use `wp_date()` or WP locale-aware month formatting for consistency with site settings.

- [x] **CR-ITEM-3.4 [Observability/testing gaps]**:
  - **Severity**: Suggestion
  - **Location**: Module-wide
  - **Description**: No logging for skipped/invalid URL records and no tests around collectors.
  - **Recommendation**: Add unit tests for each collector and integration test for admin page rendering under empty/large/error datasets.
